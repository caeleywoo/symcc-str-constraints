# SMT Solver Benchmark Constraints

This repository contains the SMT constraints generated by SymCC-STR from 5 real-world C string-processing programs, the results from running each solver on the constraints, and the scripts to replicate our results.

## Repository Structure
```
├── constraints
│   ├── mixed_constraints        # constraints generated from running in hybrid mode
│   └── string_only_constraints  # constraints generated from running in string only mode
├── results
│   ├── cvc5_results             # csv files for cvc5's performance on the constraints,
│   │                            # separated by program type and type of constraint file (sat or unsat)
│   ├── ostrich_results
│   ├── plots                    # visualized results
│   ├── z3_noodler_results
│   └── z3_results
└── run_experiments              # the scripts used to run each solver on the constraints
```

## Constraint Generation

Constraints were generated using the SymCC-STR TACAS 2026 Artifact Evaluation VM on VirtualBox. Each program was compiled with SymCC-STR's instrumented compiler (`symcc`) and run on a seed input. Two sets of constraints were generated per program:

- **Hybrid** (`set-logic ALL`) — default mode, mixes string and bitvector arithmetic
- **String-only** (`SYMCC_THEORY=str`) — restricts to pure string theory constraints

50 SAT and 50 UNSAT constraints were collected per program per theory (where available).

## Running the Benchmarks

To replicate the results, first ensure each solver (OSTRICH2, cvc5, z3, and z3 noodler) is installed, then run the scripts in the `run_experiments` folder.
